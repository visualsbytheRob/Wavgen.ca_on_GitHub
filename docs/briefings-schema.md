# Daily briefings — schema

One markdown file per news-loop run, generated by the scheduled trigger and stored at `src/diary/news/YYYY-MM-DD.md`. The Site Log (/diary/) lists them in the unified timeline alongside manual diary entries.

The loop reads the last 7 days of briefings to dedup news items — don't bulk-rename or delete files unless you're OK with possible re-coverage.

## Schema

```markdown
---
title: "Wavgen briefing — 2026-05-22"
date: 2026-05-22
sections:
  music: 4
  video: 3
  data: 5
  art: 2
---

# Wavgen briefing — 2026-05-22

One-paragraph overview of the day (what's notable, what's quiet).

## Wider context
- **World** — *[headline](url)*. One-line summary.
- **Toronto** — *[headline](url)*. One-line summary.
- **Good news** — *[headline](url)*. One-line summary.

## Music
- **[Item title](url)** — *Source, date.* One-line summary.
- ...

## Video
- ...

## Data
- ...

## Art
- ...
```

## Required frontmatter

- `title` — used by the Site Log timeline as the entry heading
- `date` — ISO date string; used for sort order in the unified timeline

The directory data file `src/diary/news/news.json` tags every entry as `"briefing"` automatically, so individual files don't need a tags field.

## Where it goes

The same content is emailed to `robmcdtv@gmail.com` on each run.

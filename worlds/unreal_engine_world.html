§ 02 — nDISPLAY CLUSTER

Multi-Node Render Cluster

nDisplay distributes Unreal Engine rendering across multiple nodes, each driving one or more displays in a synchronized cluster. A primary node orchestrates frame timing, camera frustum distribution, and genlock across the full LED volume or dome display system.

§ 03 — LIVE LINK PROTOCOL

Live Link is Unreal Engine's real-time data streaming protocol, connecting motion capture systems, face trackers, camera tracking, and custom data sources directly into the engine's animation framework with sub-frame timing accuracy.

§ 04 — IN-CAMERA VFX

LED Volume & ICVFX Workflow

In-Camera Visual Effects (ICVFX) projects a photo-real virtual environment onto a curved LED volume while the camera captures both physical foreground elements and the LED backdrop simultaneously — eliminating green screen compositing from the pipeline entirely.

§ 05 — LUMEN GLOBAL ILLUMINATION

Lumen Real-Time GI

Lumen is UE5's fully dynamic global illumination system — tracking light bounces in real-time at cinematic quality without baking. Using a combination of mesh distance fields, surface cache, and radiance cache, Lumen delivers plausible indirect lighting at interactive frame rates.

§ 06 — NANITE VIRTUALIZED GEOMETRY

Nanite Geometry Pipeline

Nanite is UE5's virtualized micropolygon geometry system — streaming and rendering only the geometry detail visible to the current camera view at pixel scale, enabling billions of polygons without artist-managed LODs or memory budget worries.

§ 07 — BLUEPRINT VISUAL SCRIPTING

Blueprint Node Graph

Blueprint is UE5's visual scripting system — a node-based graph where events, functions, and data flow are represented as connected pins. Used for gameplay logic, cinematic sequencer control, Live Link data routing, and nDisplay cluster management without writing C++.
